## Trade-offs / Justificativas

- **Plano separado em vez de task inline em Plano 01**: ordering explícito via `depends_on: [100-01]` deixa o framework executor parar entre waves se um teste do Plano 01 falhar. Não buscamos commit do bump enquanto coverage real não confirma ≥ 90%.
- **Sem ratchet 90 → 95 nesta phase**: ROI cai exponencialmente acima de 90% (líder linhas remanescentes geralmente são paths de erro raros + LLM call mocks complex). REQUIREMENTS.md já documenta 90 → 95 como Requisito Futuro adiado.
- **Margem de noise mínima**: Phase 98 usou 1.5 pp; Phase 100 segue mesma cadência se baseline ≥ 91.5%, ou ajusta comentário para refletir real margin se 90.0–91.5%. CI é deterministic per-Node-version, então variance entre runs é tipicamente <0.5 pp.

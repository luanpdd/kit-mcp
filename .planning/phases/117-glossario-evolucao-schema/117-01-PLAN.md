## Critérios de sucesso da Phase 117 (do ROADMAP.md)

1. Glossário define ≥ 15 termos canônicos PT-BR ↔ EN: linearizabilidade, consistência causal, consistência eventual, atualização perdida (lost update), distorção de escrita (write skew), leitura fantasma (phantom read), MVCC, CDC, event sourcing, fencing token, partição quente (hot partition), líder-seguidor (leader-follower), broker baseado em log, snapshot isolation, serializable snapshot isolation (SSI).
2. Glossário documenta convenção PT-BR para naming de artefatos (a partir de v1.22) com exemplos PT-BR (`evolucao-schema-compativel`) vs EN pré-v1.22 preservado (`multi-tenant-rls-hierarchy`).
3. Skill `evolucao-schema-compativel` apresenta padrão 3-passos com SQL completo: (a) `ALTER TABLE ... ADD COLUMN x text` nullable, (b) `UPDATE ... WHERE x IS NULL LIMIT 10000` em loop, (c) `ALTER TABLE ... ALTER COLUMN x SET NOT NULL` apenas após backfill verificado.
4. Skill mapeia análogos Avro/Protobuf → Postgres: rename via `CREATE OR REPLACE VIEW`, alargamento `varchar(50)→varchar(255)` seguro vs estreitamento inseguro, mudança de default em coluna em uso (deve ser 2 passos com backfill).
5. Skill documenta rolling upgrade client-side: deploy escalonado V1+V2, JWT/session compat entre versões, contratos API com versionamento de payload (campos opcionais, nunca-remover-obrigatório).
6. Ambos artefatos retornam sem erro via `mcp__kit__list_kit` (apenas o SKILL.md aparece em `listKit`; glossary é compartilhado, sem `description:`).

# Plano 103-01: Multi-window Burn-rate (1h fast + 6h slow)

## Objetivo

`/burn-rate-status` calcula burn rate dual-window (1h fast + 6h slow), aplicando precisamente a fórmula da skill `burn-rate-alerting` (lookahead/baseline fator 4×). Status enum combinado: PAGE (ambos críticos), TICKET (slow only), WARN (fast only), OK (ambos baixos), no_data.

- kit/skills/burn-rate-alerting/SKILL.md (canonical formula reference — fator 4× lookahead/baseline)

<action>
Rewrite the calculation section (Steps 3.2-3.5) of `kit/commands/burn-rate-status.md` to:

1. **Read both alert_thresholds from YAML** via regex (fast = page block, slow = ticket block):
   ```bash
   FAST_LOOKAHEAD=$(awk '/alert_thresholds:/,/^[a-z]/{if(/page:/){p=1}else if(p && /lookahead:/){print $2; exit}}' "$SLO_FILE")
   FAST_BASELINE=$(awk '/alert_thresholds:/,/^[a-z]/{if(/page:/){p=1}else if(p && /baseline:/){print $2; exit}}' "$SLO_FILE")
   FAST_MULTIPLIER=$(awk '/alert_thresholds:/,/^[a-z]/{if(/page:/){p=1}else if(p && /burn_rate_multiplier:/){print $2; exit}}' "$SLO_FILE")
   SLOW_LOOKAHEAD=$(awk '/alert_thresholds:/,/^[a-z]/{if(/ticket:/){t=1}else if(t && /lookahead:/){print $2; exit}}' "$SLO_FILE")
   SLOW_BASELINE=$(awk '/alert_thresholds:/,/^[a-z]/{if(/ticket:/){t=1}else if(t && /baseline:/){print $2; exit}}' "$SLO_FILE")
   SLOW_MULTIPLIER=$(awk '/alert_thresholds:/,/^[a-z]/{if(/ticket:/){t=1}else if(t && /burn_rate_multiplier:/){print $2; exit}}' "$SLO_FILE")
   ```

2. **Calculate fast_burn AND slow_burn** for each SLO via two `loadSnapshots()` calls with different window arguments:
   ```bash
   FAST_RESULT=$(node --input-type=module -e "
   import { loadSnapshots } from './src/core/metrics.js';
   const fast = await loadSnapshots(process.cwd(), $FAST_LOOKAHEAD_MS);
   // ... same SLI calc as before
   ")
   SLOW_RESULT=$(node --input-type=module -e "..." with $SLOW_LOOKAHEAD_MS)
   ```

3. **Combine status** per skill burn-rate-alerting fator 4× canonical:
   ```javascript
   function combinedStatus(fastBurn, fastMult, slowBurn, slowMult) {
     const fastTriggered = fastBurn >= fastMult;
     const slowTriggered = slowBurn >= slowMult;
     if (fastTriggered && slowTriggered) return 'PAGE';   // both critical → page on-call
     if (slowTriggered) return 'TICKET';                   // sustained slow erosion
     if (fastTriggered) return 'WARN';                     // fast spike alone — monitor, don't page
     if (fastBurn >= 1.0 || slowBurn >= 1.0) return 'WARN'; // ≥1× burn either window
     return 'OK';
   }
   ```

4. **Update output table** to include both windows:
   ```
   | SLO | Target | Fast (1h) | Slow (6h) | Combined | ETA exhaustão | Action |
   |---|---|---|---|---|---|---|
   | mcp-tool-availability | 99.5% | 0.42× WARN | 0.18× OK | **OK** | — | — |
   ```

5. **Cross-reference skill** in command frontmatter description + inline note pointing to `kit/skills/burn-rate-alerting/SKILL.md`.

6. **No-data logic** — if EITHER window has insufficient snapshots, status = `no_data` (conservative).

7. **Update default flags** in argument-hint to reflect dual-window: `--fast-baseline 1h --slow-baseline 6h --format table|json`.
</action>

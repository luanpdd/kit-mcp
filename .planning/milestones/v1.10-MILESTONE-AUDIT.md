## Gap Resolution

### INT-FW-V2-01 — Forense → Postmortem chain (RESOLVIDO)

**Lacuna identificada:** `/forense` escreve em `.planning/forensics/report-<ts>.md`, mas `/postmortem --from-investigation <id>` lê `.planning/investigations/<id>.md`. O `<sre_integration>` block original em `kit/commands/forense.md` sugeria literalmente `/postmortem --from-investigation <forensic-id>`, o que falharia em prática.

**Fix (commit c098ed0):**
1. `kit/commands/forense.md` — atualizou `<sre_integration>` block para sugerir `/postmortem --incident "<one-liner>"` (modo standalone) e adicionou nota distinguindo `/forense` (forensics) vs `/investigar-producao` (investigations).
2. `README.md` — atualizou quick-start example para usar `/investigar-producao` para incidents SRE → `/postmortem --from-investigation`, e separadamente `/forense` → `/postmortem --incident` para framework failures.

**Resultado:** Ambos os fluxos agora funcionam end-to-end.

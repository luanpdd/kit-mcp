## Fase 82: Web Surface Hardening

**REQs:** SEC-14-01 (CSP/XSS), SEC-14-02 (CSRF /shutdown + /publish)

**Files:**
- `src/ui/server.js` — CSP header + escape SSE payloads + auth middleware nos endpoints write
- `src/ui/lockfile.js` — campo `token` no JSON
- Possivelmente um arquivo JS estático novo (extrair JS do index.html inline)
- Tests novos em `test/integration/` (HTTP integration)

**Risco:** médio. UI server é testado mas mudanças em CSP podem quebrar consumers (browser/extension). Auth token quebra workflow de auto-spawn → browser handshake — precisa propagar token via query param.

## Fase 83: Core Filesystem Hardening

**REQs:** SEC-14-03 (reverse-sync projectRoot trust), SEC-14-04 (gate-runner tmpdir), SEC-14-05 (file-manifest verification)

**Depends on:** Phase 82 (não tecnicamente — separado por foco; ordem é convenção)

**Files:**
- `src/core/sync.js` — `handleSync` validação projectRoot
- `src/core/reverse-sync.js` — `handleReverseSync` mesma validação
- `src/core/gate-runner.js` — substituir Date.now+Math.random tmpfile por mkdtemp
- `src/core/sync.js` (path de install) — verify manifest SHA256
- Possivelmente novo helper `src/core/manifest-verify.js`
- Tests novos em `test/unit/` + `test/integration/`

**Risco:** médio-alto. Manifest verification pode ser bloqueante em dev (se kit é editado fora do hash); precisa env var `KIT_MCP_SKIP_MANIFEST_CHECK=1` para opt-out. mkdtemp deve cleanup mesmo em error paths.

## Fase 84: MCP Error Sanitization

**REQs:** SEC-14-06 (error envelope scrub + reflect.js leak)

**Depends on:** Phase 82 (separado)

**Files:**
- `src/mcp-server/index.js:281-285` — error handler central (sanitize antes de retornar ao client)
- `src/core/reflect.js` — redact tokens em error paths Anthropic
- Possivelmente novo helper `src/core/error-redaction.js`
- Tests novos em `test/unit/` (regex assertion sobre error envelope)

**Risco:** baixo-médio. Mudança localizada, mas precisa cobrir múltiplos handlers MCP (gates, sync, reverse-sync, forensics, replays). Helper `sanitizeMcpError` é central — bug aqui afeta TODA error path do MCP server.

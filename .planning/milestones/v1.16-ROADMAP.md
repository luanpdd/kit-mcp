# v1.16 ROADMAP — Performance Runtime Wave

> Fecha P1-P6 da meta-auditoria de v1.12.1 (perf runtime que ficou fora das waves segurança/tokens/drift v1.13-v1.15).

**Milestone:** v1.16 — Performance Runtime Wave
**Total de fases:** 2 (Fases 88-89)
**Estrutura:** 2 fases temáticas — concurrent I/O (P1+P3+P4) e lazy imports/optional deps (P2+P5+P6)
**Criado:** 2026-05-09 (após v1.15.0 publicado)
**Origem:** Phase 80 (v1.13) capturou TOKEN economy. P1-P6 são RUNTIME perf — disjoint set.

---

## Visão geral

A meta-auditoria de v1.12.1 identificou 6 items de perf runtime que NÃO foram tocados nas waves de v1.13-v1.15:
- v1.13: foco em CRITICAL security + token wins + drift mitigation estática
- v1.14: foco em HIGH security
- v1.15: foco em DX (auto-prevention scripts) + token wins remanescentes + CI matrix

P1-P6 são pure runtime performance — concurrent I/O + cold start. Naturalmente agrupados em 2 fases temáticas.

**Princípio:** content-zero hardening continua. Stable API preservada. Budget 6/6 deps (P5+P6 movem para optional sem dropar — fallback graceful para `npm install --omit=optional`).

---

## Fase 88: Concurrent I/O

**REQs:** PERF-16-01 (sync writes paralelos), PERF-16-02 (watch debounce), PERF-16-03 (reverse-sync walks paralelos)

**Files:**
- `src/core/sync.js` — `syncTo()` Promise.all batches=16
- `src/core/watch.js` — debounce 500ms em clearKitCache
- `src/core/reverse-sync.js` — walks (kit/+target/) em paralelo

**Risco:** médio. sync.js tocado em v1.13 (slim cap), v1.14 (manifest verify). Race conditions são risco real — testar com regression test simulando edits concorrentes.

## Fase 89: Lazy Imports & Optional Deps

**REQs:** PERF-16-04 (CLI lazy imports), PERF-16-05 (inquirer optional), PERF-16-06 (chokidar optional)

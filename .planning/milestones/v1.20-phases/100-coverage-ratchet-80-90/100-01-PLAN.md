## Trade-offs / Justificativas

- **8 test files paralelos em vez de fewer monolithic**: cada arquivo é independente; rollback de uma task não afeta outras; mais granular para review.
- **Sem novos exports `__test` em src/**: preserva Stable API v1.0+ literal. Trade: alguns helpers privados (mergeFrontmatter, kindToFolder, stripStubBoilerplate) só são testados end-to-end. Vantagem: zero risco de regressão API.
- **mcp-server tests via spawn + JSON-RPC stdin (não in-process import)**: `bin/mcp.js` chama `startStdio()` que conecta ao SDK transport globalmente; difícil unit-testar in-process sem reset do SDK state. Spawn é o pattern já usado em mcp-error-envelope.test.js.
- **upgrade-check live network teste é skipped**: registry.npmjs.org pode ser offline em CI. Cobertura de fetchLatest 100% deferida; cache paths cobrem 90% do código.

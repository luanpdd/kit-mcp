## Visão geral do milestone

v1.13 é uma **suíte de hardening interno** — diferente das suítes v1.8-v1.12 que adicionam conteúdo ao kit, esta foca em fechar gaps técnicos do projeto kit-mcp em si.

**3 vetores cobertos:**

1. **Segurança crítica** (Fase 79) — 4 vulnerabilidades CRITICAL/HIGH abertas pelo agente de segurança. `gates.run` via MCP executa shell arbitrário sem confirmação interativa; `loadReplay`/`annotateReplay` têm path traversal; publish workflow tem 2 gaps (`npm ci || npm install` fallback + skip de tests/audit antes de `npm publish`).

2. **Hooks race pattern + token economy** (Fase 80) — o fix v1.12.1 corrigiu apenas `sidecar-tool-publisher.js`; 6 outros hooks têm o mesmo bug latente. Em paralelo, capturar quick wins de tokens identificados (slim cap, dedup block hooks comentado-morto, drop CHANGELOG do tarball).

3. **Drift cleanup** (Fase 81) — 3 fontes de drift recorrente: CHANGELOG sem entries para 3 últimas releases; README com contadores hardcoded muito desatualizados; MCP server `version: '0.1.0'` enquanto package.json é v1.12.1+.

**Princípio:** todos os fixes têm origem documentada em `.planning/` (codebase/concerns.md, PRR-REPORT.md, VALIDATION.md). Cada PR/commit deve cross-referenciar a auditoria.

**v1.13 é content-zero por design** — não adiciona skills/agents/commands ao kit, só repara o framework e o package npm.

## Fase 79: Critical Security Fixes

**REQs cobertos (4):** SEC-13-01 (gates.run MCP), SEC-13-02 (replayId), SEC-13-03 (npm ci strict), SEC-13-04 (publish gates)

**Artefatos:**
- `src/mcp-server/index.js` — patch em `handleGates` (recusar via MCP sem TTY)
- `src/core/replays.js` — sanitização de `id` em `loadReplay` + `annotateReplay`
- `.github/workflows/publish.yml` — `npm ci` strict + tests + npm audit

**Risco:** baixo. Mudanças localizadas, testes existentes cobrem path feliz. Risco maior é a mudança em `handleGates` quebrar workflow legítimo de gate audit; mitigação é manter `kit gates run` CLI funcional (só remove surface MCP).

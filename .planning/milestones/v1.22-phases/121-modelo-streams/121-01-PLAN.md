## REQs cobertos

### Skill `escolha-modelo-consistencia` (MODELO-01..05)

- **MODELO-01**: Árvore de decisão linearizabilidade vs causal vs eventual com 3 exemplos canônicos por modelo (slug global, chat, feed social).
- **MODELO-02**: Uniqueness constraints distribuídos via single-leader Postgres — `UNIQUE` constraint nativo é linearizável; app-level `UPDATE+SELECT` é race; padrão correto `INSERT ... ON CONFLICT DO NOTHING RETURNING id`.
- **MODELO-03**: Análogos de total order broadcast em Postgres — `pg_current_wal_lsn()`, `pg_logical_emit_message()`, logical replication slots.
- **MODELO-04**: CAP teorema → PACELC com tabela 4 quadrantes (P+C, P+A, Normal+C, Normal+L).
- **MODELO-05**: 2PC limitações + alternativas — sagas (compensação local), transactional outbox (atomic DB write + event).

### Skill `streams-eventos-cdc` (STREAMS-01..06)

- **STREAMS-01**: Brokers AMQP/JMS-style vs log-based — tabela canônica (RabbitMQ/LISTEN/NOTIFY vs Kafka/pgmq).
- **STREAMS-02**: 3 padrões CDC em Postgres — wal2json + Realtime, pglogical → Kafka, trigger-based.
- **STREAMS-03**: Event sourcing em Postgres — tabela `events` append-only + projeções (MV ou trigger). Cross-ref ATIVO para `audit-log-multi-tenant`.
- **STREAMS-04**: Exactly-once em pgmq — dedup table com `unique(event_id)` + idempotency key + transactional outbox.
- **STREAMS-05**: 3 tipos stream join — stream-stream (janela), stream-table (CDC + atividade), table-table (merge changelogs).
- **STREAMS-06**: Log compaction — pgmq retention TTL + snapshot manual em event sourcing.
